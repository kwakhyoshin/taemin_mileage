📱
PWA Client (index.html)
Frontend · Single Page Application
단일 HTML 파일로 구성된 7,500줄+ 풀스택 PWA. 모든 UI, 로직, 상태 관리를 포함하며 모바일 최적화 반응형 디자인 적용.
  • 홈 대시보드: 마일리지, 연속기록, 오늘 날씨, 활동 현황
  • 활동 기록: 매일/매주/매달 주기별 활동 완료 체크
  • 보상 상점: 마일리지로 교환 가능한 리워드 시스템
  • 뱃지 시스템: 60개+ 자동 뱃지 + 커스텀 뱃지 (100개 SVG 아이콘 라이브러리)
  • 가족 메시지: 실시간 가족 간 메시지 교환
  • 기분 추적: 감정 기록 및 히스토리
  • 관리자 패널: PIN 보호, 활동/보상 관리, 데이터 백업
  • 달력: 활동 기록 캘린더 뷰
ES Modules CSS-in-HTML Chart.js PWA
⚙️
Service Worker (sw.js)
Background Worker · Push Handler
브라우저 백그라운드에서 실행되는 서비스 워커. Push 알림 수신, 캐싱, 오프라인 지원 담당.
  • Push Event 수신 → showNotification() 호출
  • 알림 클릭 시 앱 포커스 또는 새 창 열기
  • 기본 타이틀: '태민이 마일리지'
  • 알림 아이콘/뱃지 설정
Push API Notification API Cache API
🌐
GitHub Pages
Static Hosting · CDN
GitHub 리포지토리에서 직접 호스팅되는 정적 파일 서빙. HTTPS 자동 지원.
  • Repository: kwakhyoshin/taemin_mileage
  • Branch: main
  • 배포: GitHub Contents API를 통한 자동 배포
  • 파일: index.html, sw.js, manifest.json, logo.png
🔥
Cloud Firestore
NoSQL Database · Real-time
단일 문서(data/taemin) 기반의 전체 앱 상태 저장소. 실시간 동기화(onSnapshot) 지원.
  • pts: 현재 마일리지 포인트
  • streak / longest: 연속 / 최장 출석일
  • log[]: 전체 활동 기록 (타임스탬프, 포인트)
  • badges{}: 획득한 뱃지 목록 + 획득 시간
  • familyMessages[]: 가족 메시지 배열
  • pushDevices{}: 기기별 Push 구독 정보
  • users{}: 사용자 ID/PW 해시
  • moods{}: 현재 기분 상태
  • rwds[] / rewardLog[]: 보상 목록 / 교환 기록
taemin-mileage SDK v10.12
⚡
Cloud Functions v2
Serverless Backend · Push Server
Firestore 문서 변경 감지 트리거 기반의 서버리스 함수. Push 알림 발송 전담.
  • onFamilyMessage 새 메시지 감지 → 수신자에게 Push
  • onMoodUpdate 기분 변경 감지 → 가족에게 Push
  • onBroadcastPush 관리자 공지 → 전체 Push
  • onRewardRequest 보상 요청/승인/거절 → 관련자 Push
  • 모든 알림 제목에 [태민마일리지] prefix
  • timestamp 비교로 중복 알림 방지
Node 20 web-push VAPID
🔒
인증 시스템
Authentication · Session
Firebase Auth 미사용, 자체 구현 인증 시스템. 가족 3인 전용 간편 인증.
  • 가입: 멤버 선택 → ID/PW 생성 → SHA-512 해싱
  • 로그인: ID/PW 입력 → 해시 비교 → localStorage 세션
  • 세션 유지: taemin_current_user 키
  • 관리자 보호: 4자리 PIN (기본: 1234)
  • 비밀번호 해싱: Web Crypto API (SHA-512)
3데이터 흐름

주요 시나리오별 데이터의 이동 경로와 처리 과정

시나리오데이터 흐름프로토콜
앱 로딩 Browser → GitHub Pages (HTTPS) → index.html/sw.js 로드 → Firestore getDoc() → S 상태 초기화 → checkAuth() → renderAll() HTTPS Firestore SDK
활동 완료 사용자 탭 → S.actDone 업데이트 + pts 증가 → setDoc(DATA_DOC, S) → Firestore 저장 → checkBadges() → 새 뱃지 시 celebration 표시 Firestore SDK
메시지 전송 사용자 입력 → S.familyMessages.push() → setDoc() → Firestore onWrite trigger → Cloud Function: onFamilyMessage → web-push → Push Service → sw.js → showNotification() Firestore Web Push
보상 교환 보상 선택 → pts 차감 + rewardLog.push() → setDoc() → onRewardRequest trigger → 부모에게 승인요청 Push → 부모 승인 → 태민이에게 승인 Push Firestore Web Push
실시간 동기화 기기A setDoc() → Firestore 업데이트 → 기기B onSnapshot() 콜백 → _isSaving / _hasDirtyLocal 체크 → S = Object.assign(defaultState(), data) → renderAll() onSnapshot
Push 구독 사용자 동의 → serviceWorkerRegistration.pushManager.subscribe() → PushSubscription → S.pushDevices[user][device] 저장 → setDoc() Push API VAPID
