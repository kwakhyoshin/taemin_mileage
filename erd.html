환경 분리 규칙:
• Production: families/{familyId}, users/taemin, emailVerify/{email}
• Development: families/dev_{familyId}, users/taemin_dev, emailVerify/dev_{email}
• 레지스트리: _id_registry (prod) vs _dev_id_registry (dev)
• dayHistory 서브컬렉션: 부모 문서(families 또는 users) 아래에 위치
데이터 모드 차이:
• shared 모드: 가족 전체가 하나의 pts, acts, rwds 공유. memberData 미사용
• independent 모드: memberData[memberId]에 개인별 pts, acts, rwds, log 등 분리 저장. 전역 S.pts도 동시 갱신 (하위호환)
• 보상 동기화 (v0415e): independent 모드에서 rewardInventory/rewardRequests는 memberData[자녀]에 저장되며, Firestore 리스너가 이 변경을 감지해 다기기 로그인 자녀의 S.rewardInventory에 즉시 반영
v0415 변경 이력:
• v0415c: pushBroadcast.targetUser 필드 추가 — 관리자 테스트 푸시를 본인 계정 기기로 제한
• v0415d: pushDevices[deviceId]에 type='fcm' / fcmToken / platform 구조 추가 — Android APK FCM 수신 활성화. webpush(VAPID)와 공존
• v0415d: Cloud Functions에 activity_verify 전용 트리거 추가 (actVerifyRequests 배열 변경 감지)
• v0415e: Firestore 리스너에 memberData[자녀].rewardInventory/rewardRequests 동적 머지 로직 추가. 쿨다운 2000ms→400ms
